# Hello World Tutorial — claude-runner

This tutorial walks you from a fresh install to a fully running task in under
10 minutes.  You will:

1. Install claude-runner (from source)
2. Run `claude-runner configure`
3. Create a project folder and a simple project book
4. Run `claude-runner run` and watch Claude work
5. Inspect the output

---

## Step 0 — Prerequisites

Make sure the following are installed and working before you start.

### Node.js

```cmd
winget install OpenJS.NodeJS.LTS
```

Reopen your terminal after installing.

### Claude Code

```cmd
npm install -g @anthropic-ai/claude-code
```

Then authenticate once:

```cmd
claude
```

This opens a browser login flow.  If you have a **Claude.ai Pro or Max**
subscription, log in with your account — no API key required.  Otherwise enter
your Anthropic API key when prompted.

Close Claude Code once the login completes (`Ctrl+C` or just wait for it to
exit).

### Python 3.11+

Download from https://www.python.org/downloads/ and check "Add Python to PATH"
during installation.

> Docker Desktop is **not** required for the hello-world tutorial; we use
> the native sandbox (Claude Code runs directly on your machine).

## Step 1 — Install claude-runner

```cmd
git clone https://github.com/LunaticSodium/ClaudeRunner.git
cd claude-runner
pip install -e ".[dev]"
```

Verify the install:

```cmd
claude-runner --help
```

You should see a list of subcommands (`run`, `configure`, `validate`, …).

---

## Step 2 — Configure

```cmd
claude-runner configure
```

The wizard auto-detects your Claude Code installation and OAuth session.  A
typical session looks like:

```
[INFO]  Step 1 — Claude Code authentication
        Claude Code is installed at: C:\...\claude.cmd
        Claude Code OAuth session detected — no API key needed.
        [OK]    Using Claude Code OAuth session (Claude.ai Pro/Max account).

[INFO]  Step 2 — Push notifications (ntfy.sh)
        Choose notification method:
          1) ntfy.sh (free, instant phone/desktop push — recommended)
          2) Email (SMTP)
          3) Skip

Your choice [1/2/3]:
```

For the hello world you can press **3** (Skip) — notifications are optional.

At the end you will see:

```
[OK]    Configuration saved to C:\Users\<you>\.claude-runner\secrets.yaml
```

---

## Step 3 — Create a project directory

Create an empty folder that Claude will write its output into:

```cmd
mkdir C:\Projects\hello-claude
cd C:\Projects\hello-claude
git init
```

The `git init` is needed for the runner's git integration to track changes.

---

## Step 4 — Edit the bundled project book

Open `projects/hello_world.yaml` (included with the repo and inside the exe).
The only line you need to change is `sandbox.working_dir`:

```yaml
# projects/hello_world.yaml

name: Hello Claude
description: A simple first task to verify the whole stack is working.

prompt: |
  Please do the following three things in order:

  1. Create a file called hello.py in the current directory with the single
     line:  print("Hello from claude-runner!")

  2. Run it with:  python hello.py
     (or python3 hello.py on Mac/Linux)
     and confirm the output is correct.

  3. Write a one-line README.md that says:
     "# Hello Claude — generated by claude-runner"

  After completing all three steps, summarise what you did in one sentence.

sandbox:
  backend: native                          # no Docker needed
  working_dir: C:/Projects/hello-claude   # <-- the folder you created above

execution:
  timeout_hours: 1
  max_rate_limit_waits: 3
  resume_strategy: continue
  skip_permissions: true    # lets Claude run python without prompting you

output:
  git:
    enabled: true            # creates a git commit in working_dir when done

notify:
  on: [complete, error]
  channels:
    - type: desktop          # Windows toast notification when the task ends
```

**One thing to check before running:**

- `sandbox.working_dir` must match the folder you created in Step 3.

---

## Step 5 — Validate the project book

Check for typos before running:

```cmd
claude-runner validate projects/hello_world.yaml
```

Expected output:

```
[OK]    Project book "Hello Claude" is valid.
```

If you see a validation error, check:
- That `sandbox.working_dir` exists and is spelled correctly.
- That all indentation uses spaces, not tabs.

---

## Step 6 — Run it

```cmd
claude-runner run projects/hello_world.yaml --tui
```

The `--tui` flag (on by default) shows a live Rich terminal panel:

```
┌─ [claude-runner] Hello Claude ──────────────────────────────────┐
│ Status:    running                                               │
│ Runtime:   0h 00m 12s                                            │
│ RL waits:  0 / 3                                                 │
│ Context:   ~1 200 tokens                                         │
├──────────────────────────────────────────────────────────────────┤
│ > Creating hello.py …                                            │
│ > Running python hello.py …                                      │
│   Hello from claude-runner!                                      │
│ > Writing README.md …                                            │
└──────────────────────────────────────────────────────────────────┘
```

When Claude finishes you will see:

```
[OK]    Task completed successfully in 0h 01m 04s.
```

And a Windows toast notification pops up in the bottom-right corner.

## What just happened?

Here is what claude-runner did under the hood:

| Step | What claude-runner did |
|------|------------------------|
| Auth | Detected your Claude Code OAuth session — skipped API key injection |
| Sandbox | Used `NativeSandbox` (no Docker) — ran Claude Code in your `working_dir` |
| Launch | Spawned `claude --dangerously-skip-permissions -p <prompt>` via a Windows ConPTY |
| Monitor | Streamed Claude's output line-by-line, scanning for rate-limit signals and milestone patterns |
| Git | Detected the new/changed files, ran `git add -A && git commit -m "…"` inside `working_dir` |
| Notify | Dispatched a `complete` event → desktop toast via apprise |

## Troubleshooting

### Working directory is empty / no git history

The working directory is created automatically if it doesn't exist.
To get git tracking, run `git init` inside it before launching the task:

```cmd
git init C:\Projects\hello-claude
```

### "claude: command not found"

Claude Code is not on your PATH.  Reopen your terminal after running
`npm install -g @anthropic-ai/claude-code`, or run the full path:

```cmd
where claude
```

### Claude asks for permissions interactively

Make sure `skip_permissions: true` is in the `execution:` block of your
project book.  Without it Claude prompts you before every file write.

### Task seems stuck

claude-runner shows live output.  If nothing moves for more than 30 seconds,
Claude is probably waiting on a rate-limit.  The TUI will show a countdown
timer automatically.

### "apprise not installed — desktop notification skipped"

Install it:

```cmd
pip install apprise
```
